Web Exploitation

Web V1

Kode Python tersebut mengambil input dari pengguna melalui berbagai parameter, lalu membuat akun pengguna berdasarkan parameter yang diberikan. Pada class User, terdapat atribut is_admin. Dengan menambahkan parameter is_admin=1, kita dapat mendaftarkan akun sebagai admin secara langsung.

Register account, tambahin parameter is_admin=1.

Pada endpoint /admin/fetch terdapat celah SSRF, di mana mekanisme whitelist-nya tidak aman. Sistem hanya memeriksa apakah string daffainfo.com terdapat dalam URL yang dikirimkan. Dengan memanfaatkan kelemahan ini, kita dapat menyisipkan daffainfo.com sebagai query parameter, sehingga URL seperti http://127.0.0.1:1337/internal?daffainfo.com tetap lolos validasi dan kita bisa akses ke layanan internal.

langsung kita ngerequest ke /admin/fetch endpoint dengan url parameter http://127.0.0.1:1337/internal?daffainfo.com.

Bypass

Parameter title, akan dicheck dengan blacklist, jika aman akan masuk ke function render_template_string.

Salah satu proof of concept SSTI adalah {{7*7}} akan dievaluasi sebagai 49, akan tetapi karakter {{` dan `}} diblacklist.

Nah ternyata masih ada proof of concept lainnya, yaitu dengan menggunakan {% if 7*7==49 %} 49 {% endif %}

Waktunya ngebypass blacklistnya untuk mendapatkan RCE. Nah aku ada resource untuk dapetin payload SSTI (https://www.onsecurity.io/blog/server-side-template-injection-with-jinja2/)

Waktunya bikin solver.

Orbiterzzxx

Melakukan directory fuzzing, dan menemukan file /phpinfo.php

Ditemukan credentials buat login dan flag part 1.


Flag part-1: 345Y_P34SY

Ditemukan flag part 2 pada jwt cookie

Flag part-2: _L3M0N_

Terdapat command injection, dan ditemukan flag part 3 pada file true_flag.txt


Flag part-3: 5QU332Y

Final flag: IFEST13{345Y_P34SY_L3M0N_5QU332Y!!!}

FINAL

Web Exploitation

Web v2

User(**data) disini vulnerable, user bisa kasih parameter is_admin=1 dan bisa keregister sebagai admin.


Berhasil register dengan role admin.

Selanjutnya kita pengen ke /admin/fetch untuk melakukan SSRF, akan tetapi terdapat proteksi pada nginxnya. Ketika kita ke situ bakalan langsung di block.

Sesuai dengan source yang saya dapatkan (https://blog.1nf1n1ty.team/hacktricks/pentesting-web/proxy-waf-protections-bypass), konfigurasi itu vulnerable dimana attacker bisa ngebypass rules tersebut dengan menambahkan special character pada requestnya.

Karena nginx server menggunakan version 1.22.0, dan backendnya adalah Flask, maka karakter special itu adalah \x85 atau \xA0.

Tinggal bikin request di burpsuite, dan tambahkan \x85 setelah /admin/fetch

Berhasil ngebypass rules nginx.

Pada saat ingin melakukan ssrf ke /internal, terdapat proteksi dimana url yang kita berikan akan diparse menggunakan urlparse, dan akan dicek hostnamenya.

Kita dapat membypass proteksi itu dengan memberi \@daffainfo.com, dimana urlparse akan menganggap hostnamenya adalah daffainfo.com.

Setelah itu akan digunakan requests.get, dimana requests.get akan menganggap \@daffainfo.com sebagai url path.

Sehingga kita pake path traversal saja untuk menghilangkan /%5C@daffainfo.com itu

Dan berhasil solp 😀

PixelPlaza2

Function staticHandler berikut vulnerable terhadap path traversal

Kenapa bisa path traversal? Normalnya url itu dinormalize, tetapi jika HTTP methodnya adalah CONNECT, go ini tidak ngenormalize urlnya.


https://book.hacktricks.wiki/en/network-services-pentesting/pentesting-web/golang.html?highlight=path%20traversal%20proc#connect-method

Dengan command

kita berhasil melihat isi /etc/shadow file.

Awalnya saya mencari-cari default files, config files, /proc/self/environ untuk mencari jalan menuju fleg, akan tetapi tidak membuahkan hasil.

Pada saat saya mengunjungi /about.html, terdapat url menuju /docs/text, Nah filenya itu ga ada

Nah ternyata directory docs itu ada di directory sebelumnya.


Sehingga dapet deh flagnya
